# Obsidian Analytics Web App

## 1. Обзор проекта

Данный проект представляет собой локальное веб-приложение для анализа и визуализации данных из базы знаний Obsidian.

Система парсит структурированные заметки (Markdown + YAML frontmatter), извлекает данные, обрабатывает их, сохраняет в базу данных и отображает через веб-интерфейс.

Приложение полностью локальное и не требует деплоя.

Состоит из двух основных частей:

1. ETL (парсинг и загрузка данных)
2. Веб-интерфейс (аналитический дашборд)

## 3. Архитектура системы

### 3.1 ETL (парсер)

Шаги:

1. Чтение Markdown файлов из Obsidian
2. Парсинг YAML frontmatter
3. Извлечение данных
4. Преобразование в нормализованный формат
5. Сохранение в БД

Запуск:

* Ручной или через элемента сайта

---

### 3.2 База данных

* SQLite (один файл)
* Хранит все обработанные данные
* Является единственным источником данных для сайта

### 3.3 Веб-приложение

* Получение данных из БД
* Отображение аналитики

## 6. Архитектура проекта

```
Life_Dash/
├── data/                           # Локальные хранилища
│   └── db.sqlite                   # База данных (SQLite)
│
├── src/                            # Основной код приложения
│   ├── main.py                     # Точка сборки FastAPI (регистрация роутов)
│   │
│   ├── core/                       # Ядро системы (Shared Kernel)
│   │   ├── config.py               # Настройки (Pydantic Settings + .env)
│   │   ├── database.py             # Настройка SQLModel (engine, session getter)
│   │   ├── logger.py               # Конфигурация Loguru (форматы, файлы)
│   │   └── exceptions.py           # Кастомные ошибки (напр. ObsidianFileNotFound)
│   │
│   ├── common/                     # Общие ресурсы (Cross-cutting concerns)
│   │   ├── utils.py                # Хелперы (форматирование дат, строк)
│   │   └── validators.py           # Валидаторы данных (проверка Frontmatter, путей)
│   │
│   ├── modules/                    # БИЗНЕС-ЛОГИКА (Модули)
│   │   ├── daily_notes/            # Модуль "Ежедневные заметки"
│   │   │   ├── models.py           # SQLModel таблицы
│   │   │   ├── repository.py       # Абстракция БД (CRUD: get, create, update)
│   │   │   ├── service.py          # Бизнес-логика (аналитика, агрегация)
│   │   │   ├── schemas.py          # Pydantic-схемы (DTO) для API
│   │   │   └── router.py           # API эндпоинты (/api/notes)
│   │   │
│   │   └── [другие модули]         # finance, tasks, health...
│   │
│   ├── parser/                     # ETL-система (Obsidian -> DB)
│   │   ├── engine.py               # Основной цикл обхода файлов
│   │   ├── loaders.py              # Низкоуровневое чтение (pathlib)
│   │   ├── transformers.py         # Очистка и преобразование (regex, frontmatter)
│   │   └── tasks.py                # Async/Batch операции (массовое сохранение в БД)
│   │
│   └── web/                        # Фронтенд (Server-Side Rendering)
│       ├── router.py               # Роуты для страниц (/, /dashboard, /settings)
│       ├── static/                 # CSS (Tailwind), JS (HTMX/Alpine.js), Images
│       └── templates/              # HTML (Jinja2)
│           ├── components/         # Переиспользуемые блоки (UI Kit)
│           │   ├── card.html       # Виджет карточки
│           │   ├── chart.html      # Контейнер для Plotly
│           │   └── navbar.html
│           ├── pages/              # Целые страницы
│           │   ├── index.html
│           │   └── notes.html
│           └── layout.html         # Базовый шаблон (head, scripts)
│
├── scripts/                        # Вспомогательные скрипты
│   └── maintenance.py              # Очистка логов, бэкап БД
│
├── test/                           # Тест-кейсы
├── cli.py                          # Интерфейс командной строки (Typer)
├── logs/                           # Логи приложения (Loguru)
├── pyproject.toml                  # Зависимости Poetry
├── tailwind.config.js              # Конфиг Tailwind
└── .env                            # Секреты (путь к Obsidian, ключи)
```
