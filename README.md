# Obsidian Analytics Web App

## 1. Обзор проекта

Данный проект представляет собой локальное веб-приложение для анализа, визуализации и управления данными из базы знаний Obsidian.

Система парсит структурированные заметки (Markdown + YAML frontmatter), извлекает данные, сохраняет их в локальную базу данных и отображает через веб-интерфейс. 

**Принципы работы с данными:**
*   **Source of Truth:** Единственным источником истины является заметка в Obsidian. Данные в БД всегда актуализируются на основе содержимого файлов.
*   **Редактирование:** Изменение данных (оценок, дат, статусов) происходит только внутри Obsidian. Сайт считывает эти изменения при синхронизации.
*   **Создание:** Новые заметки можно создавать через сайт (генерация файла), после чего они попадают в БД через процесс синхронизации.

---

## 2. Источники данных (Модульная структура)

Данные разделены на независимые модули, так как каждый тип контента имеет уникальный набор полей:

*   **Books:** Автор, количество страниц, жанр, прогресс.
*   **Games:** Платформа, время прохождения, разработчик.
*   **Cinema:** Режиссер, тип (фильм/сериал), количество серий/сезонов.
*   **Daily Notes:** Сон, настроение, привычки, активность (парсинг чек-листов и списков).
*   **Weekly Notes:** Агрегированные итоги и рефлексия.

## 3. Архитектура системы

### 3.1 Синхронизация
*   **Инициация:** Обновление данных в БД запускается **только вручную через веб-интерфейс**.
*   **Гранулярность:** Возможность запустить полную синхронизацию всего или обновить конкретный модуль (например, только "Книги").
*   **Валидация и Логирование:** 
    *   При парсинге проверяется наличие обязательных полей.
    *   Все ошибки (битый YAML, отсутствие данных, неверный формат) записываются в `logs/app.log`.
    *   Файлы с критическими ошибками пропускаются, чтобы не засорять БД некорректными данными.
*   **Результат:** По завершении синхронизации в окне на сайте выводится **статистика**: сколько файлов обработано, сколько обновлено, сколько пропущено из-за ошибок.

### 3.2 ETL-система (Parser)
Процесс разделен на три независимых слоя:
1.  **Engine (Scanner):** Низкоуровневый обход директорий. Возвращает метаданные файлов (пути, даты изменения).
2.  **Transformers:** Набор специализированных классов для каждого модуля. Преобразуют сырой Markdown/YAML в объекты SQLModel.
3.  **Sync (Orchestrator):** Логика синхронизации. Сопоставляет данные сканера с БД, принимает решение об обновлении, вызывает трансформеры и обрабатывает ошибки.

### 3.3 Слой доступа к данным
1.  **Repository:** Инкапсулирует все запросы к SQLite. Отвечает за CRUD операции.
2.  **Service:** Реализует бизнес-сценарии (генерация файлов, подготовка сложных отчетов, фильтрация).

### 3.4 База данных
*   **SQLite + SQLModel:** Каждому модулю соответствует своя таблица с четкой схемой данных.

### 3.5 Веб-приложение
*   **FastAPI + HTMX:** Обеспечивает интерактивность (динамические графики, индикаторы загрузки при синхронизации) без перезагрузки страниц.

## 4. Основные функции

### 4.1 Управление данными
*   **Гибкая синхронизация:** Кнопки "Обновить всё" или "Обновить [Модуль]" на страницах разделов.
*   **Создание заметок:** Удобные формы для генерации новых MD-файлов с правильной структурой YAML.

### 4.2 Аналитика и визуализация
*   **Дашборд:** Общая сводка и ключевые метрики.
*   **Медиа:** Списки с фильтрацией, статистика оценок, графики Plotly.
*   **Время и финансы:** Анализ трат и активностей по категориям и периодам.

## 5. Технологический стек

### Backend / Parsing
*   **FastAPI / SQLModel** (Core)
*   **python-frontmatter / PyYAML** (Parsing)
*   **Loguru** (Logging)
*   **Typer** (CLI Interface)
*   **Pandas** (Data Processing)

### Frontend
*   **Jinja2** (Templates)
*   **HTMX** (Interactivity)
*   **Tailwind CSS** (Styling)
*   **Plotly.js** (Charts)

## 7. Поток данных (Data Flow)

1.  **Создание:** `Web Form` -> `Python Script` -> `Markdown File (.md)` в Obsidian Vault.
2.  **Синхронизация (Чтение):** `Obsidian (.md)` -> `Parser (Sync Engine)` -> `SQLite`.
3.  **Отображение:** `FastAPI` -> `SQLModel Query` -> `Jinja2 Template` -> `HTMX (UI)`.
4.  **Аналитика:** `SQLite` -> `Pandas/Plotly` -> `Interactive Chart`.
